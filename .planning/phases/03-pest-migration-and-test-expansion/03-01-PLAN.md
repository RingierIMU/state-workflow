# Plan 01: Pest Infrastructure Setup

## Objective

Install Pest dependencies, create `tests/Pest.php` configuration, remove the global `event()` function hack (`Helpers.php` + `funkjedi/composer-include-files`), and update `composer.json` scripts. This establishes the Pest foundation that all subsequent plans depend on.

## Context

- Current test runner: PHPUnit via `composer test` -> `phpunit`
- `tests/Fixtures/Helpers.php` overrides global `event()` function for test event capture — must be removed
- `funkjedi/composer-include-files` autoloads Helpers.php — must be removed along with it
- `TestCase.php` extends Orchestra Testbench with `ConfigTrait`, `RefreshDatabase`, `WithFaker`

## Tasks

<task type="auto">
  <name>Task 1: Install Pest dependencies</name>
  <files>composer.json, composer.lock</files>
  <action>
    Run: composer require pestphp/pest pestphp/pest-plugin-laravel --dev --with-all-dependencies
    This installs Pest 3.x (latest stable for PHP 8.3+) and the Laravel plugin.
    Pest requires PHPUnit 11, which is within the current ^10.0|^11.0 constraint.
  </action>
  <verify>
    composer show pestphp/pest — confirms package installed
    composer show pestphp/pest-plugin-laravel — confirms plugin installed
  </verify>
  <done>Both Pest packages appear in composer.json require-dev and composer.lock</done>
</task>

<task type="auto">
  <name>Task 2: Remove Helpers.php and funkjedi/composer-include-files</name>
  <files>tests/Fixtures/Helpers.php, composer.json</files>
  <action>
    1. Delete tests/Fixtures/Helpers.php (the global event() override)
    2. Run: composer remove funkjedi/composer-include-files --dev
    3. Remove the "extra.include_files" section from composer.json if it still exists
    4. Run: composer dump-autoload
  </action>
  <verify>
    test ! -f tests/Fixtures/Helpers.php — file deleted
    grep -c "funkjedi" composer.json returns 0 — package removed
    grep -c "include_files" composer.json returns 0 — extra section cleaned
  </verify>
  <done>Helpers.php deleted, funkjedi package removed, no include_files in composer.json</done>
</task>

<task type="auto">
  <name>Task 3: Create tests/Pest.php configuration</name>
  <files>tests/Pest.php</files>
  <action>
    Create tests/Pest.php with:
    ```php
    <?php

    use Ringierimu\StateWorkflow\Tests\TestCase;

    uses(TestCase::class)->in(__DIR__);
    ```
    This binds TestCase (which includes ConfigTrait, RefreshDatabase, WithFaker) as the base for all Pest test files. The $this->user property from setUp() will be available in all test closures.
  </action>
  <verify>
    test -f tests/Pest.php — file exists
    grep "uses(TestCase::class)" tests/Pest.php — binding present
  </verify>
  <done>tests/Pest.php exists with correct TestCase binding</done>
</task>

<task type="auto">
  <name>Task 4: Update composer.json test script</name>
  <files>composer.json</files>
  <action>
    Update "scripts.test" from "phpunit" to "vendor/bin/pest" in composer.json.
  </action>
  <verify>
    grep '"test"' composer.json | grep -q 'pest' — script updated
  </verify>
  <done>composer test now runs vendor/bin/pest</done>
</task>

<task type="auto">
  <name>Task 5: Verify Pest boots correctly</name>
  <files></files>
  <action>
    Run: vendor/bin/pest --version
    This confirms Pest is installed and executable.
    Then run: vendor/bin/pest --list-tests 2>&1 | head -20
    This confirms Pest can discover existing PHPUnit tests (Pest is backwards-compatible with PHPUnit test classes).
  </action>
  <verify>
    vendor/bin/pest --version outputs a version string
    vendor/bin/pest runs without fatal errors (existing PHPUnit tests should still pass under Pest runner)
  </verify>
  <done>Pest executes, discovers existing tests, no configuration errors</done>
</task>
